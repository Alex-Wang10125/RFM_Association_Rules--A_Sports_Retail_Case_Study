# 客户分群与产品关联分析：棒球用品电商综合业务分析

一项基于 SQL + Python 的电商客户分群、产品关联以及综合业务分析项目。初始数据来源于阿里天池竞赛的棒球用品订单明细。我使用 SQL 完成数据初步处理与聚合，Python 部分采用 RFM + K-Means 聚类、Apriori 算法进行机器学习，并完成数据可视化。最终识别出 0.19% 的高价值客户并发现了 5 个产品社区，优化了业务策略。

## 1. 项目背景

这是我的第一个个人求职项目，制作初衷是向本土 HR 证明自己掌握 SQL 和 Python 数据分析工具，并且有一定的商业直觉。

作为数据分析行业的新人，我也希望看到这个项目的全球同行能指点一二，因此我额外制作了 README 和报告的英文版本。

与一些单纯聚焦数据挖掘的分析/机器学习项目不同，本项目更强调业务视角。我把自己设想为一家真实电商公司的数据部门负责人，从模拟电商业务的角度分析问题，并基于真实世界的特点设计可落地的业务策略。

## 3. 如何运行

本项目分为 SQL 和 Python 两部分。分析报告经过多个阶段形成，脚本之间存在承接关系，后续脚本依赖前一脚本的输出。

**推荐执行顺序：**  
`01_db_setup.sql` → `02_db_preprocess.sql` → `01_exploratory_analysis.py` → `02_ml_modeling.py`

所有脚本均使用我个人计算机上的本地目录。如果您将数据下载到自己的本地数据目录，请手动调整脚本内的输入/输出目录设置，以确保正确读取文件。

**主要工具和版本：**

- MySQL 8.4
    
- MySQL Workbench 8.0.46
    
- Python 3.14
    
- Visual Studio Code 1.11

## 5. 分析方法

本项目的最终目标是通过机器学习解决业务问题。

前三个脚本侧重于数据预处理、描述性统计和探索性分析，帮助我们理解数据特征并为机器学习做准备。在此阶段，我采用了数据聚合和常见的描述统计方法，统计了电商领域大部分基础业务指标。

机器学习脚本则利用处理好的数据进行客户分层与产品关联分析。具体方法包括：

- **RFM + K-Means 聚类**进行客户价值评估与分层；
    
- **Apriori 算法**构建购物篮矩阵，挖掘强关联规则；
    
- **Louvain 算法**构建产品共现网络，发现产品社区。
    

## 6. 主要发现

（1）高价值客户占比 **0.19%** ，沉睡客户占比 **99.81%** ；但由于产品的低复购属性，当前电商模式应视为**忠诚度模式**，不必急于获取新客户。  
（2）核心产品组合（`541-棒球手套` → `530-棒球手套`，`540-棒球手套` → `529-棒球手套`）提升度高达 **16.05**，强烈建议捆绑销售。

## 7. 业务建议

基于数据分析，提出 **5 类可落地**的业务建议：

1. **高价值客户维护：** 通过集中采购折扣、VIP 专属服务，便利核心客户，增强用户黏性。
    
2. **沉睡客户唤醒：** 借助邮件推送、内容营销、售后活动等方式，重新激活客户需求。
    
3. **全新捆绑套餐：** 依据强关联规则，推出两类主题套餐（如“掌握全局手套套装”、“袋中寒光球棒套装”）。
    
4. **算法交叉推荐：** 在商品详情页动态展示“经常一起购买”的商品组合。
    
5. **线上货架优化：** 将关联度高的产品（如球棒与球棒袋）在页面中相邻摆放，提升发现几率。
